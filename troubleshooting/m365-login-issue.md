# Microsoft 365 Login Issue – Troubleshooting Guide

## 📝 Scenario Overview
A user reports they cannot sign in to Microsoft 365 services (Outlook, Teams, SharePoint, or the Office portal). The error message varies but commonly includes: *"Your account or password is incorrect"*, *"We couldn’t sign you in"*, or *"Account not found."*

## 🔍 Initial Checks
- Confirm correct email address is being used.
- Verify internet connection.
- Ask if the user recently changed their password.
- Test login on multiple devices.

## ❓ Clarifying Questions
1. Are you signing in with your full email?
2. Are you receiving any specific error codes?
3. Did you recently reset your password?
4. Can you sign in using a private/incognito window?
5. Are you on a company or personal device?

## 🛠️ Troubleshooting Steps
1. Verify the account exists in Microsoft 365 Admin Center.
2. Reset the password and require password change at next login.
3. Check for account lockout.
4. Test login from a private browser or another device.
5. Clear cached credentials (Credential Manager + browser cookies).
6. Check MFA configuration.
7. Confirm the user has a valid Microsoft 365 license.
8. Check Microsoft 365 Service Health Dashboard for outages.

## 🧠 Root Cause Examples
- Password expired or incorrectly set.
- Cached credentials causing failures.
- MFA misconfiguration.
- Missing or incorrect license.
- Account temporarily locked.
- Browser token corruption.

## ✅ Resolution Summary
- Reset password.
- Clear cached credentials.
- Reconfigure MFA.
- Assign or correct licenses.
- Confirm successful login.

## 📌 Preventive Actions
- Add password verification to onboarding.
- Keep MFA backup methods updated.
- Review license assignments regularly.
- Avoid saving outdated credentials in browsers.
